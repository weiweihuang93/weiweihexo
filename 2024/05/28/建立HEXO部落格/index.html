發表於2024-05-28|HEXSCHOOL
|閱讀量:

只要一到指令，瞬間建立一則部落格！

常用Hexo指令

* hexo new ‘文章名稱’
* hexo server
* hexo generate - 用於public
* hexo clean - 清除暫存
* hexo deploy - 部屬

快速認識Hexo目錄結構，建立全站設定檔案config.yml

客製化頁面 About me

全域網站接入客製選單

  • 指令 hexo new page 'About'

  • hexo-theme-landscape 樣式連結 連結

  • Install via git:

    git clone --depth 1 https://github.com/hexojs/hexo-theme-landscape themes/landscape

themes資料夾 自動新增 landscape

  • _config.yml 有Home/Archives 橫幅內容，再加入About

安裝Git，準備推到免費空間

讓 VSCode 綁定你的 GitHub 帳號，準備部署網站

環境-在 GitHub 上開一個網頁空間！

免費空間 - 用 GitHub Pages 來部署你的部落格

Hexo + GitHub Pages 官方部署說明 連結

1. 安裝 hexo-deployer-git.
    

2. 清空 _config.yml 的現有資料，並新增以下組態:
    deploy:
        type: git
        repo: https://github.com/<username>/<project>
        # example, https://github.com/hexojs/hexojs.github.io
        branch: gh-pages

3. 執行 hexo clean && hexo deploy
4. 瀏覽 <GitHub 用戶名>.github.io 檢查你的網站能否運作。

註：_config.yml檔案共有兩處須更改，除了上述的部分，還有

5. URL - Set your site url here. For example, if you use GitHub Page, set url as

認識 layout.ejs 主樣版

增加 fullPage 指定滿版變數

  • About / index.md 加入 fullPage:true
  • layout.ejs 加入 !page.fullPage &&

增加 CSS 變成滿版樣式

  • class="<%= page.fullPage === true ? 'fullPage' : '' %>">

這個表達式的作用是根據page.fullPage的值來動態地決定是否給該元素添加”fullPage”這個class。
如果page.fullPage的值為true，則該元素會有”fullPage”這個class，否則不添加任何class。

<style>
    .fullPage #main{
    width: 100%;
    float: none;
    }
        </style>

隱藏部落格區塊方法

.fullPage .article-meta,
.fullPage .article-header,
.fullPage .article-footer{
  display: none;
}

讓首頁的部落格頁面移動到 blog 路由網址

_config.yml
index_generator:
path: '/blog'
per_page: 10
order_by: -date

themes/landscape/_config.yml
menu:
首頁: /
部落格: /blog
關於我: /About
